## 1. 执行摘要

### 一句话定位

> **DBCheck — 面向混合数据库基础设施的智能运维操作系统。不只是巡检，而是数据库全生命周期的"自动驾驶"平台。**

### 核心价值主张

中国数据库市场正经历前所未有的变革——信创国产化替代进入深水区，企业IT环境从单一 Oracle/MySQL 演变为 **MySQL + PostgreSQL + TiDB + DM8 + KingbaseES + GBase 8s + YashanDB + Oracle** 的混合架构。数据库种类激增带来了运维复杂度爆炸式的增长。

DBCheck 从**巡检切入**，已覆盖 10 种数据库，下一步是成为：

| 阶段 | 产品形态 | 对标 |
|------|---------|------|
| 现在 | 数据库巡检工具（CLI + Web UI + Docker） | SolarWinds + 国产化 |
| 1年后 | 混合数据库运维中台（管理中心） | Datadog + 信创适配 |
| 3年后 | 数据库基础设施 AI 运维操作系统 | Splunk + AIGC |

### 为什么现在是最佳时机？

1. **信创窗口期（2025–2027）**：党政/金融/央企/教育全面替换，混合数据库管理是刚需
2. **AI 平民化**：Ollama 让本地 AI 诊断零成本落地，AIGC 与运维的结合是 2026 年最大技术红利
3. **开源生态成熟**：DBCheck 已有开源用户基础（GitHub 170+ Stars，Docker Hub 88+ Pulls），Product-Led Growth 模式已初步验证

## 7. 技术架构演进

### 7.1 当前架构（v2.6）

```
浏览器 ──→ Flask Web UI ──→ SQLite (状态/用户/基线)
                │
        ┌───────┼───────┐
        │       │       │
      MySQL  Oracle  GBase ...
    (pymysql)(oracledb)(JDBC)
```

### 7.2 目标架构（v3.0+ 中台化）

```
┌──────────────────────────────────────────────────────────┐
│                    管理控制台 (Web UI)                     │
│   多租户 · RBAC · SSO · Grafana 面板 · 告警中心           │
└────────────┬─────────────────────────────────────────────┘
             │ gRPC / REST API
┌────────────▼─────────────────────────────────────────────┐
│                    DBCheck Server (核心引擎)              │
│  任务调度 · 规则引擎 · AI 诊断 · 报表生成 · 消息推送       │
│  存储：PostgreSQL + Redis + MinIO                        │
└────────────┬─────────────────────────────────────────────┘
             │ mTLS
     ┌───────┼───────┬───────┬───────┐
     ▼       ▼       ▼       ▼       ▼
  Agent-1 Agent-2 Agent-3 Agent-4 Agent-N
  (MySQL) (PG)  (Oracle)(GBase) (TiDB)
  · 轻量二进制 (<10MB)
  · 支持断线缓存
  · 自动注册发现
  · 可过跳板机/堡垒机
```

### 7.3 关键技术选型

| 组件 | 当前 | 目标 | 原因 |
|------|------|------|------|
| Web 框架 | Flask | FastAPI（企业版） | 异步性能、自动 OpenAPI 文档 |
| 数据库 | SQLite | PostgreSQL | 多租户隔离、并发写入 |
| 消息队列 | 无 | Redis / RabbitMQ | 异步任务解耦 |
| 通信协议 | HTTP | gRPC + HTTP | Agent-Server 高性能通信 |
| 对象存储 | 本地文件系统 | MinIO / S3 | 报告/知识库集中存储 |
| AI 引擎 | Ollama（本地调用） | Ollama + vLLM 可选 | 多模型支持、GPU 加速 |
| 容器编排 | Docker Compose | K8s Helm Chart | 企业级高可用 |
