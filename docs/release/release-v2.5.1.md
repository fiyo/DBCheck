# DBCheck v2.5.1 发布：实时监控仪表盘，让数据库状态一目了然

## 什么是 DBCheck？

**DBCheck** 是一款开源的跨平台数据库自动化巡检工具，支持 **MySQL、PostgreSQL、Oracle、SQL Server、达梦 DM8、TiDB、IvorySQL** 七种主流关系型数据库。

它能为你的数据库做什么？

- **自动化健康巡检**：150+ 条增强规则，覆盖性能、安全、配置、连接、锁等多个维度，自动生成 Word 巡检报告
- **AI 智能诊断**：接入本地 Ollama 大模型，基于巡检数据生成个性化优化建议
- **历史趋势分析**：多次巡检数据自动汇聚，生成指标趋势图，对比发现潜在问题
- **服务器巡检**：全面检查 CPU、内存、磁盘、网络等系统资源
- **分享链接**：一键生成在线分享链接，无需登录即可查看报告
- **定时巡检 + 通知**：Cron 表达式配置定时任务，完成后自动发送邮件或 Webhook 通知
- **RAG 知识库**：上传数据库文档，AI 诊断时自动检索相关知识增强分析准确性

简单说，**DBCheck 就是 DBA 的自动化巡检助手**——把枯燥的重复检查工作交给工具，让你专注于真正需要关注的问题。

项目地址：[https://github.com/fiyo/dbcheck](https://github.com/fiyo/dbcheck)
官网：[https://dbcheck.top](https://dbcheck.top)

## v2.5.1 新功能：实时监控仪表盘

之前的 DBCheck 已经能帮你做**定期巡检**——但有些问题需要**实时感知**。

比如：

- 慢查询突然暴增，但下一次巡检还要等几小时？
- 某个数据库的连接数悄悄逼近上限，巡检报告还没来得及生成？
- 活跃会话中存在阻塞，你希望第一时间发现而不是等报告？

**v2.5.1 新增的实时监控仪表盘**，就是为解决这些问题而生的。

### 实时慢查询监控

![实时慢查询监控](snapshot/monitor_slow.png)

跨数据源的慢查询实时监控，核心特性：

| 特性 | 说明 |
|------|------|
| 多数据源聚合 | 所有已配置数据源的 Top SQL 自动聚合到一张表格 |
| 数据库类型识别 | 根据数据源名称自动识别并显示彩色标签 |
| 严重度分级 | 🔴 高危 / 🟡 中等 / 🟢 低危，一眼识别风险等级 |
| 灵活筛选排序 | 按耗时、次数、时间范围、关键字多维筛选 |
| CSV 导出 | 一键导出当前数据，UTF-8 BOM 编码，Excel 直接打开无乱码 |
| 自动刷新 | 5-60 秒可配置轮询间隔，带实时倒计时 |

**设计亮点**：
- 四张概览卡片（监控数据源数、慢查询数、平均耗时、最慢 SQL）带渐变图标和上下文子标签
- 执行耗时智能着色——红色表示严重、橙色表示警告、绿色表示正常
- 严重度使用圆点指示器而非纯文字，视觉层级更清晰

### 活跃连接监控

![活跃连接监控](snapshot/monitor_conn.png)

全数据源连接状态实时可视化，核心特性：

| 特性 | 说明 |
|------|------|
| 连接使用率柱状图 | 每个数据源的当前连接数 / 最大连接数，按使用率分色 |
| 12 小时热力图 | 按小时聚合的历史连接趋势，5 级色彩快速识别使用规律 |
| 活跃会话 TOP 10 | 实时展示最活跃会话，含用户、状态、时长、SQL 预览 |
| 阻塞检测 | 概览卡片直接高亮阻塞会话和受影响数据源 |
| 空闲连接追踪 | 监控最长空闲连接持续时间 |

**设计亮点**：
- 左右分栏布局——左侧连接柱状图 + 热力图，右侧活跃会话表
- 热力图从传统 HTML 表格升级为 flex 网格，按小时聚合，更直观
- 连接柱状图颜色阈值明确：绿色（<50%）→ 黄色（50-80%）→ 红色（>80%）

### 使用方式

1. 在 Web UI 侧边栏点击 **📡 实时监控**
2. 点击页面标题右侧的 **▶ 启动** 按钮开始监控
3. 设置轮询间隔（默认 10 秒，可调 5-60 秒）
4. 通过筛选条件精确查看关注的数据源或 SQL
5. 需要暂停时点击 **⏹ 停止** 即可

无需安装任何 Agent，无需额外配置，**配置好数据源就能直接用**。

## v2.5.1 UI 改进

除了新功能，这次还做了一系列 UI 优化：

- **启动/停止按钮**：带 SVG 图标的渐变色胶囊按钮，放置在页面标题右侧，不再隐藏在筛选栏中

- **连接条形图标签**：宽度从固定 100px 改为自适应 140-240px，长数据源名称不再截断
- **整体风格**：与设计稿对齐的深色主题仪表盘风格

## 如何升级

**源码部署**：
```bash
git pull origin main
pip install -r requirements.txt
python web_ui.py
```

**PyInstaller 打包**：
```bash
git pull origin main
pyinstaller dbcheck.spec
cd dist && ./dbcheck
```

访问 [https://dbcheck.top](https://dbcheck.top) 获取最新版本。
