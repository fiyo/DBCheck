# DBCheck v2.4.0 发版说明 — 数据源管理中心

## 版本概要

**版本号：** v2.4.0
**发版日期：** 2026-05-03
**核心新增：** 数据源管理模块（Pro），统一管理多数据库实例，支持分组管理、连接测试、CSV 导入导出

---

## 新增功能

### 1. 数据源管理模块

新增完整的数据源管理功能（Pro 版本），支持统一管理所有数据库实例：

| 文件 | 功能 |
|------|------|
| `pro/instance_manager.py` | 数据源实例管理器：CRUD、连接测试、CSV 导入导出、分组管理 |

**核心功能：**

| 功能 | 说明 |
|------|------|
| 多数据库支持 | MySQL / PostgreSQL / Oracle / SQL Server / DM8 / TiDB |
| 实例信息 | 自定义标签、分组、端口、用户名 |
| Oracle 专属 | 服务名/SID 配置、SYSDBA 特权连接 |
| 连接测试 | 一键测试数据库连接，实时返回结果 |
| 分组管理 | 按业务/环境分组，支持自定义颜色标签 |
| CSV 导入导出 | 批量导入/导出数据源配置 |

### 2. Web UI 数据源管理页面

在 Web UI 中新增「数据源管理」导航页（🗄️ 图标）：

- **添加数据源：** 选择数据库类型 → 填写连接信息 → 保存
- **编辑数据源：** 修改任意字段，密码可选择性更新
- **删除数据源：** 支持二次确认，防止误删
- **连接测试：** 对已保存的数据源进行连接验证
- **分组筛选：** 按分组过滤显示，快速定位目标实例
- **批量导入：** CSV 文件批量导入数据源
- **批量导出：** 一键导出所有数据源为 CSV

**新增 API 接口：**

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/pro/datasources` | GET | 列出所有数据源 |
| `/api/pro/datasources` | POST | 添加数据源 |
| `/api/pro/datasources/<id>` | GET | 获取数据源详情 |
| `/api/pro/datasources/<id>` | PUT | 更新数据源 |
| `/api/pro/datasources/<id>` | DELETE | 删除数据源 |
| `/api/pro/datasources/<id>/test` | POST | 测试数据源连接 |
| `/api/pro/datasources/import` | POST | CSV 批量导入 |
| `/api/pro/datasources/export` | GET | CSV 导出 |
| `/api/pro/groups` | GET | 列出所有分组 |
| `/api/pro/groups` | POST | 创建分组 |
| `/api/pro/groups/<name>` | PUT | 更新分组 |
| `/api/pro/groups/<name>` | DELETE | 删除分组 |

### 3. 首页布局优化

全新设计的首页布局，更加现代化和清晰：

**新增区域：**
- **核心功能区：** 2 列卡片网格展示核心功能，突出"快速巡检"入口
- **数据库支持展示：** 6 列卡片网格展示所有支持的数据库类型

**响应式设计：**
- 大屏幕：6 列数据库展示 + 2 列功能卡片
- 中等屏幕：3 列数据库 + 2 列功能
- 小屏幕：2 列数据库 + 单列功能

### 4. Oracle 连接优化

**Oracle 默认用户名改为 system：**
- 新建数据源选择 Oracle 时，用户名输入框自动填入 `system`
- 新建巡检向导同样适用

**服务名标签优化：**
- 标签从"服务名 (Oracle)"改为"服务名/SID (Service Name)"
- 添加 Oracle 小标签，与新建巡检向导风格一致

**SYSDBA 复选框：**
- 新增"以 SYSDBA 身份连接"复选框
- 与服务名输入框之间增加适当间距

## 升级指南

### 已有用户升级步骤

1. **拉取代码**
   ```bash
   git pull origin main
   ```

2. **重启 Web UI**
   ```bash
   python web_ui.py
   ```

3. **验证**
   - 访问 Web UI → 点击左侧导航「🗄️ 数据源管理」
   - 添加一个数据源进行测试
   - 创建分组并测试分组筛选功能

### 数据源管理使用指南

#### 添加数据源

1. 进入「数据源管理」页面
2. 点击「添加数据源」按钮
3. 选择数据库类型
4. 填写连接信息：
   - **标签：** 用于识别的名称
   - **主机地址：** 数据库服务器 IP
   - **端口：** 根据数据库类型自动填充
   - **用户名：** 根据数据库类型自动填充
   - **密码：** 数据库密码
   - **Oracle 专属：** 服务名/SID、「以 SYSDBA 身份连接」
5. 点击保存

#### 批量导入

1. 先导出现有数据源作为模板
2. 按模板格式编辑 CSV 文件
3. 在数据源管理页面点击「批量导入」
4. 选择 CSV 文件上传
